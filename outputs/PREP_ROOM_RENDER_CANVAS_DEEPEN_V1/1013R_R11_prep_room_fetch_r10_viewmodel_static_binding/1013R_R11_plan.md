# 1013R_R11：R9 fetch R10 ViewModel 静态绑定规划

## 定位

R11 不继续堆静态文字，也不接模型/provider/runtime。

R11 的任务是：复制 R9 当前可见页为新副本，在不改 R36、不改主壳正式入口的前提下，让新副本从 R10 只读 ViewModel 读取当前课对象、步骤状态、备课正文、右侧任务状态和边界 flags。

```text
stage_id=1013R_R11_PREP_ROOM_R9_FETCH_R10_VIEWMODEL_STATIC_BINDING
source_page=1013R_R9_real_textbook_prep_page_sync
source_viewmodel=1013R_R10_PREP_ROOM_SINGLE_LESSON_VIEWMODEL_READONLY_ENDPOINT
visible_task_object=三年级第二单元《多彩的世界》第1课《色彩的渐变》
R36_modified=false
provider_called=false
model_called=false
formal_apply_performed=false
```

## 去风险决定

R9 页面里存在历史遗留的 `main-shell-visible-hydration-1013L-R11` 静态 payload，其中包含旧课题《色彩的感觉》。R11 不以此作为数据来源。

R11 去风险方式如下：

1. R9 原文件不改，继续作为已 PASS 基线。
2. R11 新副本中明确禁用或旁路旧的 `main-shell-visible-hydration-1013L-R11` 逻辑。
3. R11 新副本不调用旧的全文 text walker 替换策略。
4. R11 只接受 R10 endpoint 返回的字段：
   - `task_object`
   - `prep_view_patch.lesson_tree`
   - `prep_view_patch.current_lesson`
   - `courseware_screen_patch`
   - `source_matrix`
   - `runtime_actions`
   - `boundary`
5. R11 validator 必须检查新副本中 R11 主绑定代码没有读取旧 payload id。
6. R11 validator 必须检查 R11 主绑定代码没有调用 `replaceVisibleText(payload)` 作为主绑定方式。
7. R11 validator 必须检查 R10 渲染数据中不出现《色彩的感觉》。

这个风险点因此不再作为实施阻塞项，而是转为 R11 的硬边界和验收条件。

## 允许范围

R11 允许：

1. 复制 R9 HTML 为 R11 新副本。
2. 保留 R9 当前视觉，包括 2K 阅读轴、步骤块、状态圈、深灰绿色文字。
3. 新增 R11 fetch 绑定脚本。
4. fetch R10：

```text
GET /api/prep-room/single-lesson-viewmodel/g3_u2_l1_color_gradient
```

5. file:// 打开时使用 `window.__XIAOBEI_API_BASE__ || "http://127.0.0.1:8083"` 拼接本地 API。
6. fetch 成功时，把 R10 ViewModel 映射到页面已有模型。
7. fetch 失败时保留 R9 fixture，不让页面空白。
8. 输出 R11 报告、结果 JSON 和 validator。

## 禁止范围

R11 禁止：

```text
改 R36
改主壳正式入口
回到 R22/O_R2/M_R5
接模型/provider
写数据库/记忆/向量/飞书
formal apply
把 AI 生成资料当标准依据
复制 Claude/OpenClaw runtime
把旧 1013L_R11 hydration payload 当数据源
用全文 text walker 作为主绑定策略
```

## 实施步骤

### Step 1：复制副本

创建：

```text
outputs/PREP_ROOM_RENDER_CANVAS_DEEPEN_V1/1013R_R11_prep_room_fetch_r10_viewmodel_static_binding/
  prep_room_render_canvas_deepen_v1_1013R_R11_prep_room_fetch_r10_viewmodel_static_binding.html
```

源文件：

```text
outputs/PREP_ROOM_RENDER_CANVAS_DEEPEN_V1/1013R_R9_real_textbook_prep_page_sync/prep_room_render_canvas_deepen_v1_1013R_R9_real_textbook_prep_page_sync.html
```

### Step 2：新增 R11 绑定脚本

新增独立脚本 id：

```text
script-1013R-R11-fetch-r10-viewmodel-static-binding
```

核心常量：

```text
STAGE_ID=1013R_R11_PREP_ROOM_R9_FETCH_R10_VIEWMODEL_STATIC_BINDING
VIEWMODEL_ROUTE=/api/prep-room/single-lesson-viewmodel/g3_u2_l1_color_gradient
API_BASE=window.__XIAOBEI_API_BASE__ || "http://127.0.0.1:8083"
```

### Step 3：绑定策略

R11 不做全页面文字搜索替换。

R11 只做定点映射：

```text
R10 task_object -> 当前对象 / 右侧任务状态
R10 prep_view_patch.current_lesson.sections -> 备课正文区
R10 prep_view_patch.current_lesson.process_steps -> 教学过程区
R10 courseware_screen_patch -> 大屏草稿/课件预览区
R10 runtime_actions -> 可确认动作区
R10 boundary -> 只读边界标记
```

如果现有 DOM 没有稳定挂载点，R11 可新增小范围 `data-1013r-r11-*` 挂载点，但不得重写整页壳。

### Step 4：失败降级

fetch 失败时：

```text
data-1013r-r11-bound=false
window.__PREP_ROOM_R11_VIEWMODEL__=null
页面继续显示 R9 fixture
显示只读轻提示：R10 ViewModel 未连接，当前为 R9 静态预览
```

不能清空正文，不能回旧课题，不能触发正式写入。

### Step 5：2K 视觉核对

保留 R9 视觉要求：

```text
步骤块保留
步骤块不是说明卡片
STEP 文字深灰绿色 #314f46
状态圈跟随最长步骤文字后对齐
2K 下阅读轴不飘
不增加一大段可见说明
```
